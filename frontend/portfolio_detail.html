Flask 기반 웹 포트폴리오

Flask 프레임워크를 기반으로 백엔드와 프런트엔드를 구조적으로 분리하여 설계한
개인 포트폴리오 웹사이트입니다.

프로젝트 개요

프로젝트명: 김서현의 PORTFOLIO
개발기간: 2025.07 ~ 2025.08
사용 기술: Flask, Python, HTML5, CSS3, JavaScript, Render, Vercel

Flask의 라우팅 구조, 정적 파일 관리, 템플릿 구성 방식을 직접 설계하여 개인 포트폴리오 웹사이트를 제작했습니다.
이후 Render(백엔드)와 Vercel(프런트엔드)을 활용해 실제 서비스 환경에서 작동하는 형태로 배포했습니다.

구현 과정

- Flask 프로젝트 구조 설계

백엔드와 프런트엔드를 분리하여 관리하기 위해 backend와 frontend 디렉토리로 구성했습니다.
Flask는 API 라우팅과 서버 역할을 담당하고, HTML, CSS, JS는 frontend 폴더 내에서 정적 리소스로 관리했습니다.

- app.py 라우팅 및 서버 구성

  • Flask로 페이지 라우팅 및 정적 파일 제공 구현
  • 템플릿 경로와 정적 경로 분리로 파일 관리 효율화
  • 간단한 API 엔드포인트를 통해 동적 데이터 연결 가능성 확보

- 프런트엔드 구성 (HTML / CSS / JS)

  • base.html: 공통 레이아웃 정의 (왼쪽 고정 섹션 + 오른쪽 콘텐츠 영역)
  • index.html: 자기소개 및 기본 정보 페이지
  • projects.html: 프로젝트 카드 및 상세 페이지 링크 구성
  • contact.html: 연락처 및 외부 링크 정리

주요 특징

구분설명
Flask 기반 백엔드Flask를 이용해 라우팅 및 정적 파일 제공 기능 구현

분리형 배포Render(백엔드) + Vercel(프런트엔드)로 실제 서비스 환경 구축
수동 배포GitHub push 시 Vercel에서 수동으로 빌드 및 반영

결과 및 배포

Flask 애플리케이션은 Render에, 정적 프런트엔드는 Vercel에 배포하여 실제 웹 서비스처럼 구현되도록 설정했습니다. 이를 통해 Flask 기반 웹 애플리케이션의 개발부터 배포까지의 전 과정을 경험했습니다.
